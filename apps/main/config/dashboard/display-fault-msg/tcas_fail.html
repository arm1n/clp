TCAS FAIL


TERRAIN FAIL on both MFD1 FIM TASK 34-43-00-810-802 (TR-FIM)

NOTE: The TCAS FAIL message is shown when the traffic alert and collision avoidance system (TCAS) processor senses a condition that follows:

  • Internal malfunction
  • No continuity through the TCAS top or bottom directional antenna
  • Does not receive valid roll or pitch data from the attitude and heading reference systems (AHRS1 and AHRS2)
  • Does not receive valid heading data from AHRS1 and AHRS2
  • Does not receive valid radio altitude data from the radio altimeters (RA1 and RA2)
  • TA valid pin programming malfunction
  • RA valid pin programming malfunction
  • Does not receive valid data from the set mode S transponder (ATC1 or ATC2)
  • Does not receive valid control data from audio and radio control display unit 1 (ARCDU1) and ARCDU2
  • Does not receive valid data from the air data computer (ADC1 or ADC2) that is set for altitude reporting
  • Gillham compare senses an encoding altitude difference
  • Mode S transponder and distance measuring equipment (DME) suppression
  • TCAS configuration module has malfunctioned

The MFD can also show a weather radar STAB FAIL message.

NOTE: The primary fault indication for the TCAS is shown in the TCAS display area of the RCDU TCAS/ATC particular page.

NOTE:If you set the ATC1 or ATC2 to ON before you do the test, make sure that the Flight Management System 1 (FMS1) and FMS2 is ON and initialized. If you do not do this, it is possible that the related ATC1 or ATC2 will fail and the TCAS test will also fail. This is not a malfunction condition.









NOTE:
  • Ensure that no IOP 2 messsage is displayed on ED.
  • Reset TCAS (F1) CB on Avionics CB panel - Wait several minutes.

    • NOTE: More than one power reset might be needed and is allowed in this special case. The system requires 2 min after each reset until power up is completed. This time shall be given to the system before further attempts may be made.

  • If TCAS is still failed,
      • check fault code on TCAS processor or via maintenance terminal for further troubleshooting.
